1. 모듈이란?

모듈 : 컨트롤러, 서비스, 필터, 지시자를 담는 그릇

하나의 angualrJs app은 하나의 모듈을 지정할 수 있다.

대부분의 앱은 메인 메서드를 가지고 있어 파트별로 인스턴스화 되고 연결되어 있다.

angularjs는 메인 메서드를 가지고 있지 않은 대신 모듈을 선언하여 사용한다.

관련된 기능을 하나로 묶어 다른 코드와 결합도를 줄이고 재 사용성을 높이기 위해 사용한다.

  • 모듈 별로 쪼개서 프로세스를 쉽게 이해할 수 있다
  • 재사용이 가능한 모듈로 패키징할 수 있다
  • 필요한 모듈은 언제든 로딩하여 사용할 수 있다.
  • 단위 테스트가 수월하고 end-to-end 테스팅도 가능한다
angular.module("모듈이름", ["사용할 모듈"])

angular.module 함수를 이용해 모듈을 만들면 모듈 인스턴스가 반환

모듈 인스턴스 메서드 (v1.4.11.기준)

모듈 메서드 설명
module.config(configFunction) 모듈이 로딩될 때 호출되며 config 함수에 해당 익명 함수로 서비스를 설정할 수 있다.
moduel.constant(name, object) 상수 등록
module.controller(name, constructor) 컨트롤러 등록
module.direcetive(name, directiveFactory) 지시자 등록
module.factory(name, providerFuction) 서비스를 팩토리 형태로 등록
module.filter(name, filterFactory) 필터 등록
module.provider(name, providerType) 서비스를 제공하는 프로바이더를 등록
module.run(initializationFn) 앱 초기화 함수 등록. 모든 모듈 등록이 완료된 후 초기화 함수 실행
module.service(name, constructor) 서비스 등록
module.value(name, object) 객체 등록
module.decorator(name, decoFn(function, array)) 데코레이션 등록
module.animation(name, animationFactory(function)) 애니메이션 등록
